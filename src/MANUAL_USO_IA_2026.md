# Manual de Uso Eficiente de IA para Projetos (2026)

## 1. Princípio central
IA é cara em contexto, não em inteligência.
Envie apenas o necessário. Pense antes de chamar a IA. Trabalhe por deltas, não por dumps de código.

## 2. Fluxo padrão de trabalho
1. Definir problema em 1 frase
2. Identificar arquivo/função afetada
3. Formular saída desejada
4. Chamar IA com contexto mínimo
5. Testar localmente
6. Voltar apenas com erro real

## 3. Template universal de prompt
**Tarefa:**
(1 frase)

**Contexto mínimo:**
(arquivo/função relevante)

**Estado atual:**
(o que já funciona)

**Problema:**
(o que precisa mudar)

**Saída desejada:**
(o que a IA deve retornar)

## 4. Regras de eficiência
- Nunca envie o projeto inteiro
- Nunca peça análise geral se o problema é local
- Separe design de implementação
- Use a IA para gerar, não para lembrar
- Mantenha arquivos locais de contexto (`TODO`, `ARCHITECTURE`, `STATE`)

## 5. Quando usar IA
- Gerar boilerplate
- Explorar soluções
- Revisar arquitetura
- Criar testes
- Refatorações grandes

## 6. Quando não usar IA
- Debug simples
- Leitura do próprio código
- Decisões triviais
- Ajustes pequenos

## 7. Estrutura de sessão eficiente
**Sessão ideal:**
70% trabalho local
30% IA

**Sessão ruim:**
90% IA
10% pensamento

## 8. Arquivos recomendados no projeto
- `ARCHITECTURE.md`
- `TODO.md`
- `STATE.md`
- `NOTES.md`

## 9. Perguntas que a IA deve fazer
- Qual arquivo específico?
- Qual função específica?
- Qual comportamento esperado?
- O que já foi testado?

## 10. Regra final
IA é um copiloto.
Você continua sendo o sistema de controle.
